# VoxClone - 需求 1：用户认证与组织选择

## 📝 需求描述

实现完整的用户认证流程，包括登录、注册、组织选择和组织切换，支持多租户协作场景。

---

## 🎯 核心功能

### 1.1 用户认证（Authentication）
- 用户可以通过邮箱/密码或社交账号登录
- 新用户可以注册账户
- 会话管理（自动登录、登出）

### 1.2 组织管理（Organization）
- 用户可以创建组织（团队）
- 用户可以在多个组织间切换
- 每个组织有独立的语音库和生成记录

### 1.3 访问控制（Authorization）
- 未认证用户只能访问登录/注册页面
- 认证用户必须选择组织才能访问主功能
- 组织数据完全隔离

### 5. 获取用户信息（在服务端）

```tsx
import { auth } from '@clerk/nextjs/server';

export async function someServerFunction() {
  const { userId, orgId, sessionId } = await auth();

  if (!userId) {
    // 未认证
    return redirect('/sign-in');
  }

  if (!orgId) {
    // 未选择组织
    return redirect('/org-selection');
  }

  // 已认证且选择了组织，可以访问业务数据
  // userId 和 orgId 用于数据隔离
}
```

### 6. 获取用户信息（在客户端）

```tsx
'use client';

import { useUser, useOrganization } from '@clerk/nextjs';

export function UserInfo() {
  const { user } = useUser();  // 当前用户
  const { organization } = useOrganization();  // 当前组织

  return (
    <div>
      <p>用户: {user?.fullName}</p>
      <p>组织: {organization?.name}</p>
    </div>
  );
}
```

## 🔐 权限控制策略

### 1. 页面级保护

在需要认证的页面中，使用中间件或服务端组件检查：

```tsx
// src/app/(dashboard)/page.tsx
import { auth } from '@clerk/nextjs/server';
import { redirect } from 'next/navigation';

export default async function DashboardPage() {
  const { userId, orgId } = await auth();

  if (!userId) {
    redirect('/sign-in');
  }

  if (!orgId) {
    redirect('/org-selection');
  }

  // 渲染受保护的内容
  return <DashboardView />;
}
```

### 2. API 路由保护

```ts
// src/app/api/some-route/route.ts
import { auth } from '@clerk/nextjs/server';

export async function GET(request: Request) {
  const { userId, orgId } = await auth();

  if (!userId || !orgId) {
    return Response.json({ error: 'Unauthorized' }, { status: 401 });
  }

  // 处理业务逻辑
}
```

### 3. tRPC 过程保护

```ts
// src/trpc/init.ts
import { auth } from '@clerk/nextjs/server';
import { initTRPC } from '@trpc/server';

const t = initTRPC.create();

// 需要认证的过程
export const authProcedure = t.procedure.use(async ({ next }) => {
  const { userId } = await auth();

  if (!userId) {
    throw new TRPCError({ code: "UNAUTHORIZED" });
  }

  return next({ ctx: { userId } });
});

// 需要组织的过程
export const orgProcedure = t.procedure.use(async ({ next }) => {
  const { userId, orgId } = await auth();

  if (!userId) {
    throw new TRPCError({ code: "UNAUTHORIZED" });
  }

  if (!orgId) {
    throw new TRPCError({ code: "FORBIDDEN", message: "Organization required" });
  }

  return next({ ctx: { userId, orgId } });
});
```

## 📊 数据流

```
1. 用户访问应用
   ↓
2. 检查认证状态（Clerk）
   ↓
3. 未认证 → 显示登录/注册页面
   ↓
4. 用户登录/注册
   ↓
5. 检查组织选择
   ↓
6. 未选择组织 → 显示组织选择页面
   ↓
7. 用户创建或选择组织
   ↓
8. 访问主应用（Dashboard）
   ↓
9. 所有 API 请求携带 userId 和 orgId
   ↓
10. 数据库操作按 orgId 隔离
```

    e.preventDefault();

      identifier: email,
      password: password,
    });

  };

  // 自定义表单...

### 2. 组织邀请
邀请其他用户加入组织：

```tsx
import { useOrganization } from '@clerk/nextjs';

const { organization } = useOrganization();

const inviteUser = async (email: string) => {
  await organization?.inviteMember({ emailAddress: email });
};
```

# VoxClone 项目学习大纲（15节课）

## 项目简介
VoxClone 是一个开源的 ElevenLabs 替代品，提供 AI 文本转语音（TTS）和零样本语音克隆功能。项目基于现代 Web 技术栈：Next.js 16、React 19、TypeScript、Tailwind CSS、shadcn/ui、Prisma、Clerk、tRPC、React Query、Cloudflare R2、Modal（GPU 推理）、Polar（计费）和 Sentry（错误监控）。

本大纲旨在通过 15 节循序渐进的课程，带你从零开始理解并掌握该项目的所有技术要点，最终能够独立开发和部署类似的全栈应用。

---

## 课程安排

### 第 1 课：项目概览与环境搭建
**学习目标**：了解项目背景、功能和技术栈，完成开发环境准备。
- 项目功能介绍（文本转语音、语音克隆、多租户、计费）
- 技术栈深度解析（Next.js 16、React 19、TypeScript、Tailwind CSS 等）
- 开发工具安装（Node.js、Git、IDE 推荐）
- 项目克隆与依赖安装
- 环境变量配置（`.env.example` 解读）
- 运行开发服务器

**实践任务**：成功启动本地开发环境，访问 http://localhost:3000。

**涉及技术**：Node.js、npm、Git、环境变量。

---

### 第 2 课：Next.js 16 与 React 19 基础
**学习目标**：掌握 Next.js App Router 和 React 19 的新特性。
- Next.js App Router 目录结构解析
- 布局（Layout）、页面（Page）、加载（Loading）组件
- React 19 的新特性（Actions、useOptimistic、useFormStatus 等）
- 服务端组件（Server Components）与客户端组件（Client Components）
- 流式渲染（Streaming）与 Suspense
- 项目中的 App Router 实践（`src/app` 结构）

**实践任务**：创建一个简单的服务端组件和客户端组件，理解数据流。

**涉及技术**：Next.js 16、React 19、服务端组件。

---

### 第 3 课：样式与组件库（Tailwind CSS + shadcn/ui）
**学习目标**：掌握 Tailwind CSS 原子化样式和 shadcn/ui 组件库的使用。
- Tailwind CSS v4 新特性与配置（`tailwind.config.ts`）
- CSS 变量与主题定制
- shadcn/ui 组件库的安装与配置（`components.json`）
- 常用组件（Button、Card、Form 等）的使用与定制
- 响应式设计与移动端适配
- 项目中的样式体系分析

**实践任务**：使用 shadcn/ui 组件构建一个简单的表单页面。

**涉及技术**：Tailwind CSS、shadcn/ui、CSS 变量。

---

### 第 4 课：数据库设计与 Prisma ORM
**学习目标**：掌握数据建模、Prisma 迁移和数据库操作。
- 数据库选型（PostgreSQL）与连接配置
- Prisma 数据模型定义（`schema.prisma`）
- 关系建模（用户、组织、语音、生成记录等）
- 迁移文件的生成与应用（`prisma migrate`）
- Prisma Client 的使用（查询、事务、关联）
- 数据库种子脚本（`scripts/seed-system-voices.ts`）

**实践任务**：设计一个简单的数据模型，并执行迁移和种子操作。

**涉及技术**：PostgreSQL、Prisma、数据建模。

---

### 第 5 课：认证与多租户（Clerk）
**学习目标**：集成 Clerk 实现用户认证和多租户组织。
- Clerk 功能介绍（用户管理、组织、Webhooks）
- Clerk 与 Next.js 集成（中间件、Provider）
- 保护路由（`(dashboard)` 布局）
- 获取用户会话信息（`auth()`）
- 组织（Organizations）管理与数据隔离
- 项目中的认证流程分析（登录、注册、组织选择）

**实践任务**：实现一个需要认证的页面，并获取当前用户信息。

**涉及技术**：Clerk、Next.js 中间件、JWT。

---

### 第 6 课：后端架构（tRPC + React Query）
**学习目标**：理解 tRPC 类型安全的 API 架构和 React Query 状态管理。
- tRPC 核心概念（路由器、过程、上下文）
- tRPC 与 Next.js App Router 集成（`api/trpc/[trpc]/route.ts`）
- 定义查询（Query）和变更（Mutation）
- React Query 的集成（缓存、失效、乐观更新）
- 错误处理与输入验证（Zod）
- 项目中的 tRPC 路由器结构分析

**实践任务**：创建一个简单的 tRPC 过程，并在前端调用。

**涉及技术**：tRPC、React Query、Zod。

---

### 第 7 课：文件存储（Cloudflare R2 / AWS S3）
**学习目标**：集成对象存储服务，管理音频文件。
- 对象存储服务选型（R2、S3）
- AWS SDK v3 的使用（上传、下载、预签名 URL）
- 文件上传流程（前端 Dropzone、后端处理）
- 预签名 URL 实现安全访问（`api/audio/[generationId]/route.ts`）
- 存储桶策略与 CORS 配置
- 项目中的音频存储实现分析

**实践任务**：实现一个文件上传接口，并生成预签名 URL 供前端下载。

**涉及技术**：Cloudflare R2、AWS SDK、预签名 URL。

---

### 第 8 课：TTS 模型部署（Modal + Chatterbox）
**学习目标**：部署 GPU 推理服务，集成 Chatterbox TTS 模型。
- Modal 平台介绍（Serverless GPU）
- Chatterbox TTS 模型简介（零样本语音克隆）
- Python 服务部署（`chatterbox_tts.py`）
- Modal  secrets 与环境变量管理
- 容器挂载 R2 存储桶
- 生成类型安全的 API 客户端（`npm run sync-api`）

**实践任务**：部署一个简单的 Modal 函数，并测试 API 调用。

**涉及技术**：Modal、Python、Docker、GPU 推理。

---

### 第 9 课：文本转语音功能实现（UI 层）
**学习目标**：构建 TTS 功能的用户界面。
- 页面布局与组件分解（`features/text-to-speech`）
- 表单设计（文本输入、参数调节、语音选择）
- 状态管理（React Hook Form + Zod）
- 实时参数调整与预览
- 响应式布局（移动端抽屉）
- 语音选择器（组合框、网格）

**实践任务**：实现一个文本输入表单，包含参数调节和语音选择。

**涉及技术**：React Hook Form、Zod、shadcn/ui 组件。

---

### 第 10 课：文本转语音功能实现（逻辑层）
**学习目标**：实现 TTS 生成的后端逻辑和前后端交互。
- 生成请求流程（前端 → tRPC → Modal API）
- 音频文件处理（存储、元数据保存）
- 生成状态管理（加载、错误、成功）
- 实时进度反馈（可选）
- 错误处理与重试机制
- 项目中的 `generate` 过程分析

**实践任务**：实现一个完整的 TTS 生成流程，包括前端调用、后端处理、存储和状态更新。

**涉及技术**：tRPC、Modal API、文件存储、状态管理。

---

### 第 11 课：音频播放器（WaveSurfer.js）
**学习目标**：集成 WaveSurfer.js 实现波形可视化音频播放。
- WaveSurfer.js 核心概念（实例、插件、事件）
- 波形绘制与自定义样式
- 播放控制（播放/暂停、跳转、速度调节）
- 音频下载与分享
- 响应式播放器（移动端适配）
- 项目中的音频播放器组件分析

**实践任务**：集成 WaveSurfer.js，实现一个支持波形显示和基本控制的音频播放器。

**涉及技术**：WaveSurfer.js、HTML5 Audio、Canvas。

---

### 第 12 课：语音克隆功能实现
**学习目标**：实现零样本语音克隆功能。
- 语音克隆原理简介（零样本学习）
- 语音样本录制（前端 `recordrtc`）
- 样本上传与处理（格式验证、元数据提取）
- 克隆请求发送与状态跟踪
- 克隆语音的存储与管理
- 项目中的语音克隆流程分析

**实践任务**：实现语音录制、上传和克隆功能。

**涉及技术**：recordrtc、音频处理、Modal API。

### 第 13 课：多租户组织与数据隔离

- 组织模型设计（Clerk Organizations + 数据库关联）

**实践任务**：实现一个组织相关的查询，确保数据隔离。

**涉及技术**：Clerk Organizations、Prisma、tRPC 中间件。

## 学习建议

1. **循序渐进**：按照课程顺序学习，每节课完成实践任务。
2. **动手实践**：在理解代码的基础上，尝试修改和扩展功能。
3. **查阅文档**：每个技术栈都有官方文档，遇到问题优先查阅。
4. **参考源码**：结合项目源码学习，理解实际应用场景。
5. **加入社区**：遇到问题可以在项目 GitHub 讨论区或相关技术社区提问。

## 项目介绍

### 什么是 VoxClone？
VoxClone 是一个开源的 ElevenLabs 替代品，提供 AI 驱动的文本转语音（TTS）和零样本语音克隆功能。这意味着你可以：
1. **文本转语音**：将任意文字转换为自然、富有表现力的语音
2. **语音克隆**：只需10秒的语音样本，即可克隆任何人的声音
3. **多租户支持**：团队协作，数据隔离
4. **按量计费**：基于使用量的灵活计费模式

### 核心功能
- **文本转语音**：支持调整创造力、多样性、表达力和流畅度参数
- **零样本语音克隆**：无需训练模型，直接克隆声音
- **20个内置语音**：预置系统语音，涵盖12个类别和5种语言环境
- **波形音频播放器**：WaveSurfer.js 可视化，支持播放、暂停、跳转和下载
- **多租户架构**：通过 Clerk Organizations 实现团队访问和完整数据隔离
- **基于用量的计费**：通过 Polar 实现按字符计费，可配置价格
- **生成历史**：浏览和重放历史生成记录，保留语音元数据
- **完全响应式**：移动优先设计，支持底部抽屉、紧凑控件和自适应布局

## 技术栈深度解析

VoxClone 采用了现代化的全栈技术栈，以下是各个技术的作用：

### 前端框架
- **Next.js 16**：React 元框架，提供 App Router、服务端组件、流式渲染等
- **React 19**：最新版 React，支持 Actions、useOptimistic 等新特性
- **TypeScript**：类型安全的 JavaScript 超集，提高代码可靠性

### 样式与 UI
- **Tailwind CSS v4**：实用优先的 CSS 框架，原子化样式
- **shadcn/ui**：基于 Radix UI 的可定制组件库
- **CSS 变量**：主题定制和动态样式

### 后端与数据
- **tRPC**：类型安全的 API 框架，无需手动定义类型
- **React Query**：服务器状态管理，支持缓存、失效、乐观更新
- **Prisma**：现代 ORM，类型安全的数据库操作
- **PostgreSQL**：关系型数据库，存储用户、语音、生成记录等

### 认证与多租户
- **Clerk**：用户认证和管理，支持组织（多租户）
- **JWT**：JSON Web Tokens，安全的身份验证机制

### 文件存储
- **Cloudflare R2**：S3 兼容的对象存储，存储音频文件
- **AWS SDK v3**：与云存储服务交互的 SDK

### AI 与部署
- **Modal**：Serverless GPU 平台，部署 Chatterbox TTS 模型
- **Chatterbox TTS**：开源的零样本语音克隆模型
- **Python**：TTS 模型部署语言

### 计费与监控
- **Polar**：基于用量的计费平台
- **Sentry**：错误监控和性能追踪
- **Stripe**：支付处理（通过 Polar 集成）

### 开发工具
- **ESLint**：代码质量检查
- **Prettier**：代码格式化
- **Husky**：Git hooks

## 环境变量配置

### 1. 复制环境变量文件
```bash
# 复制示例环境变量文件
cp .env.example .env
```

### 2. 理解环境变量
打开 `.env` 文件，以下是各个变量的作用：

#### 计费相关（Polar）
```env
POLAR_ACCESS_TOKEN=          # Polar API 访问令牌
POLAR_SERVER=sandbox         # Polar 服务器（sandbox 或 production）
POLAR_PRODUCT_ID=            # Polar 产品 ID
POLAR_METER_VOICE_CREATION=voice_creation     # 语音创建计量器名称
POLAR_METER_TTS_GENERATION=tts_generation     # TTS 生成计量器名称
POLAR_METER_TTS_PROPERTY=characters           # TTS 计量属性
```

#### 错误监控（Sentry）
```env
SENTRY_AUTH_TOKEN=           # Sentry 身份验证令牌
```

#### TTS 模型（Chatterbox）
```env
CHATTERBOX_API_KEY=          # Chatterbox API 密钥（任意强随机字符串）
CHATTERBOX_API_URL=          # Modal 部署的 Chatterbox API URL
HF_ACCESS_TOKEN=             # Hugging Face 访问令牌（下载模型权重）
```

#### 文件存储（Cloudflare R2）
```env
R2_ACCOUNT_ID=               # R2 账户 ID
R2_ACCESS_KEY_ID=            # R2 访问密钥 ID
R2_SECRET_ACCESS_KEY=        # R2 秘密访问密钥
R2_BUCKET_NAME=              # R2 存储桶名称
R2_TOKEN=                    # R2 令牌
```

#### 应用配置
```env
APP_URL=http://localhost:3000    # 应用 URL
```

#### 认证（Clerk）
```env
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=    # Clerk 公钥
CLERK_SECRET_KEY=                     # Clerk 密钥
# 其他 Clerk 路由配置...
```

#### 数据库
```env
DATABASE_URL=                # PostgreSQL 数据库连接字符串
```

### 3. 开发环境简化配置
对于本地开发，我们可以先配置最小化的环境变量：

```env
# 跳过环境验证（开发时使用）
SKIP_ENV_VALIDATION=true

# 应用 URL
APP_URL=http://localhost:3000

# Clerk 配置（需要先创建 Clerk 应用）
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_xxx
CLERK_SECRET_KEY=sk_test_xxx
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up
NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL=/
NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL=/

# 数据库（本地 PostgreSQL）
DATABASE_URL="postgresql://postgres:password@localhost:5432/resonance"

# 其他变量暂时留空，后续课程中配置
```

### 4. 常见问题解决

#### 端口占用
如果 3000 端口被占用，可以指定其他端口：
```bash
npm run dev -- -p 3001
```

#### 依赖安装失败
```bash
# 清除缓存并重新安装
rm -rf node_modules package-lock.json
npm cache clean --force
npm install
```

#### TypeScript 错误
```bash
# 检查 TypeScript 配置
npx tsc --noEmit
```

## 常见问题 FAQ

### Q: 为什么访问页面显示 Clerk 错误？
A: 需要先配置 Clerk 环境变量。可以在 [Clerk Dashboard](https://dashboard.clerk.com) 创建测试应用获取密钥。

### Q: 数据库连接失败怎么办？
A: 确保 PostgreSQL 已安装并运行，或者使用 `SKIP_ENV_VALIDATION=true` 跳过数据库验证。

### Q: 依赖安装太慢怎么办？
A: 可以尝试使用淘宝镜像：
```bash
npm config set registry https://registry.npmmirror.com
```

### Q: 遇到奇怪的 TypeScript 错误？
A: 尝试删除 `node_modules` 和 `package-lock.json` 后重新安装。

# 第6课：数据失效与重新获取

## 6.1 回顾与概述

在第5课中，我们学习了 useMutation 的用法，知道了如何修改数据。

本节课我们将学习一个非常重要的概念：**数据失效（Invalidation）**，这是保证客户端数据和服务端同步的关键机制。

## 6.2 为什么要数据失效？

### 6.2.1 问题场景

```typescript
// 1. 页面加载时，获取用户列表
const { data: users } = useQuery(trpc.users.getAll.queryOptions());

// 2. 用户删除了一个用户
const deleteMutation = useMutation(trpc.users.delete.mutationOptions());

// 3. 删除成功后，列表还是旧数据！
// 因为 Query 还在使用缓存，不会自动刷新
```

### 6.2.2 解决方案

```typescript
const deleteMutation = useMutation(
  trpc.users.delete.mutationOptions({
    onSuccess: () => {
      // 删除成功后，让 users 列表失效
      queryClient.invalidateQueries({
        queryKey: trpc.users.getAll.queryKey(),
      });
    },
  }),
);
```

### 6.2.3 invalidateQueries 的作用

```
调用 invalidateQueries 后：
1. 标记该 queryKey 对应的数据为"过期"
2. 自动触发新的请求获取最新数据
3. 所有使用该 queryKey 的组件都会更新
```

## 6.3 invalidateQueries 基本用法

### 6.3.1 精确匹配

```typescript
// 只让 ['users'] 失效
queryClient.invalidateQueries({
  queryKey: ['users'],
});
```

### 6.3.2 前缀匹配

```typescript
// 让所有以 ['users'] 开头的都失效
queryClient.invalidateQueries({
  queryKey: ['users'],
  exact: false,  // 默认 false，启用前缀匹配
});

// 失效的 keys：
// - ['users']
// - ['users', '123']
// - ['users', { status: 'active' }]
```

### 6.3.3 精确匹配（默认）

```typescript
// 只有完全匹配的才失效
queryClient.invalidateQueries({
  queryKey: ['users', '123'],
  exact: true,
});

// 只失效 ['users', '123']
// 不失效 ['users']
// 不失效 ['users', '456']
```

### 6.3.4 predicate 函数

```typescript
// 自定义过滤条件
queryClient.invalidateQueries({
  predicate: (query) => {
    // 让所有 users 相关的都失效
    return query.queryKey[0] === 'users';
  },
});
```

## 6.4 项目中的实际用法

### 6.4.1 案例1：删除声音后刷新列表

```typescript
// src/features/voices/components/voice-card.tsx
const deleteMutation = useMutation(
  trpc.voices.delete.mutationOptions({
    onSuccess: () => {
      toast.success("Voice deleted successfully");
      queryClient.invalidateQueries({
        queryKey: trpc.voices.getAll.queryKey(),
      });
    },
  }),
);
```

**解析**：
- `trpc.voices.getAll.queryKey()` 返回 `['trpc', 'voices.getAll']`
- 调用后，该 key 对应的缓存失效
- React Query 自动发起新请求获取最新列表

### 6.4.2 案例2：创建声音后刷新列表

```typescript
// src/features/voices/components/voice-create-form.tsx
const form = useAppForm({
  onSubmit: async ({ value }) => {
    try {
      await createMutation.mutateAsync({
        name: value.name,
        file: value.file!,
        // ...
      });

      toast.success("Voice created successfully!");
      queryClient.invalidateQueries({
        queryKey: trpc.voices.getAll.queryKey(),
      });
      form.reset();
    } catch (error) {
      // 错误处理
    }
  },
});
```

### 6.4.3 案例3：刷新多个相关的 Query

```typescript
// 如果有多个相关的数据需要刷新
onSuccess: () => {
  // 刷新用户列表
  queryClient.invalidateQueries({ queryKey: ['users'] });
  
  // 刷新用户统计
  queryClient.invalidateQueries({ queryKey: ['users', 'stats'] });
  
  // 刷新权限列表
  queryClient.invalidateQueries({ queryKey: ['permissions'] });
}
```

## 6.5 queryKey 的重要性

### 6.5.1 queryKey 决定缓存

```typescript
// 两个组件使用相同的 queryKey，共享缓存
const Component1 = () => {
  // queryKey: ['users']
  useQuery(trpc.users.getAll.queryOptions());
};

const Component2 = () => {
  // queryKey: ['users']
  useQuery(trpc.users.getAll.queryOptions());
};

// Component1 修改数据后 invalidateQueries
// Component2 会自动刷新！
```

### 6.5.2 tRPC 生成的 queryKey

```typescript
// tRPC 自动生成 queryKey
trpc.voices.getAll.queryKey()
// → ['trpc', 'voices.getAll']

trpc.voices.getAll.queryKey({ query: 'search' })
// → ['trpc', 'voices.getAll', { query: 'search' }]

trpc.generations.getById.queryKey({ id: '123' })
// → ['trpc', 'generations.getById', { id: '123' }]
```

### 6.5.3 正确的 queryKey 选择

```typescript
// ❌ 错误：只刷新特定参数
queryClient.invalidateQueries({
  queryKey: trpc.voices.getAll.queryKey({ query: 'abc' }),
});
// 只会刷新 query='abc' 的缓存

// ✅ 正确：刷新整个 voices.getAll
queryClient.invalidateQueries({
  queryKey: trpc.voices.getAll.queryKey(),
});
// 会刷新所有 voices.getAll 的缓存（无论参数）
```

## 6.6 setQueryData vs invalidateQueries

### 6.6.1 invalidateQueries（推荐）

```typescript
// 刷新数据，重新请求
queryClient.invalidateQueries({
  queryKey: ['users'],
});
```

**优点**：
- 简单直接
- 总是获取最新数据
- 适用于大多数场景

### 6.6.2 setQueryData（乐观更新）

```typescript
// 直接更新缓存，不请求
queryClient.setQueryData(['users'], (old) => {
  return old.filter(user => user.id !== deletedId);
});
```

**优点**：
- 响应更快（立即更新，无需等待服务器）
- 适用于需要即时反馈的场景

**缺点**：
- 需要手动维护数据一致性
- 失败时需要回滚

### 6.6.3 项目中只用 invalidateQueries

```typescript
// 项目中的做法：简单直接
onSuccess: () => {
  queryClient.invalidateQueries({
    queryKey: trpc.voices.getAll.queryKey(),
  });
}
```

**原因**：
- 数据量不大，网络请求快
- 避免乐观更新的复杂性
- 代码更简单，不易出错

## 6.7 其他刷新方式

### 6.7.1 refetchQueries

```typescript
// 立即刷新指定 Query，不等下次访问
queryClient.refetchQueries({
  queryKey: ['users'],
});
```

### 6.7.2 fetchQuery

```typescript
// 手动获取数据并更新缓存
const data = await queryClient.fetchQuery({
  queryKey: ['users'],
  queryFn: fetchUsers,
});
```

### 6.7.3 手动设置数据

```typescript
// 手动更新缓存
queryClient.setQueryData(['user', '123'], {
  id: '123',
  name: 'Tom',
  updated: true,
});
```

## 核心原理：URL 即状态

nuqs 的核心思想是让 **URL 成为 React 状态的单一来源（Single Source of Truth）**。

### 1. 状态同步机制

```
┌──────────────┐     window.history      ┌──────────────┐
│  React State │◄──────────────────────►│     URL      │
│  (useState)  │                        │ (searchParams)│
└──────────────┘                        └──────────────┘
       ▲                                         │
       │              useQueryState               │
       │              (Hook 包装)                 │
       └────────────────────────────────────────┘
```

当使用 `useQueryState` 时：
1. 读取 URL 参数 → 转换为对应类型 → 返回 React 状态
2. 设置 React 状态 → 序列化为字符串 → 更新 URL

### 2. 实现细节

```typescript
// 简化版的 useQueryState 实现
function useQueryState<T>(
  key: string,
  parser: Parser<T>
): [T | null, (value: T | null) => void] {
  // 1. 获取当前 URL 参数
  const searchParams = useSearchParams();
  const value = searchParams.get(key);

  // 2. 通过 parser 解析
  const parsed = parser.parse(value);

  // 3. 创建 setter
  const setValue = (newValue: T | null | ((prev: T) => T)) => {
    // 如果是函数，获取新值
    const resolved = typeof newValue === 'function'
      ? newValue(parsed)
      : newValue;

    // 处理 null：清除 URL 参数
    const serialized = resolved === null
      ? null
      : parser.serialize(resolved);

    // 更新 URL
    const newParams = new URLSearchParams(window.location.search);
    if (serialized === null) {
      newParams.delete(key);
    } else {
      newParams.set(key, serialized);
    }

    window.history.pushState(null, '', `?${newParams}`);
  };

  return [parsed, setValue];
}
```

## Parser 解析器系统

### 1. Parser 接口定义

```typescript
interface Parser<T> {
  // 从 URL 字符串解析为目标类型
  parse(value: string | null): T | null;

  // 序列化为 URL 字符串
  serialize(value: T): string | null;

  // 类型
  type: T;
}

// 示例：parseAsInteger 实现
const parseAsInteger = () => ({
  parse: (value: string | null) => {
    if (!value) return null;
    const parsed = parseInt(value, 10);
    return isNaN(parsed) ? null : parsed;
  },
  serialize: (value: number) => String(value),
  type: null as number | null
});
```

### 2. 内置 Parser 列表

| Parser | URL 示例 | 解析后类型 |
|--------|---------|-----------|
| `parseAsString` | `?name=foo` | `string` |
| `parseAsInteger` | `?page=3` | `number` |
| `parseAsFloat` | `?price=9.99` | `number` |
| `parseAsBoolean` | `?dark=true` | `boolean` |
| `parseAsIsoDateTime` | `?date=2025-01-01` | `Date` |
| `parseAsArrayOf` | `?tags=a&tags=b` | `string[]` |
| `parseAsJson` | `?filters={...}` | `object` |

### 3. 组合 Parser

```typescript
// parseAsArrayOf 内部实现
const parseAsArrayOf = <T>(parser: Parser<T>): Parser<T[]> => ({
  parse: (values: string | null) => {
    if (!values) return null;
    // URL: ?tags=a&tags=b 会被解析为 ['a', 'b']
    const arr = values.split(',');
    return arr.map(v => parser.parse(v)).filter(Boolean);
  },
  serialize: (values: T[]) => {
    return values.map(v => parser.serialize(v)).join(',');
  },
  type: [] as T[]
});
```

## 批量更新 (Batching)

### 问题

当同时更新多个参数时，如果每次都单独更新 URL，会导致多次浏览器历史记录。

```
// ❌ 错误：触发多次历史记录
setLat(42);      // history.pushState #1
setLng(12);      // history.pushState #2

// ✅ 正确：批量更新，一次历史记录
setCoordinates({ lat: 42, lng: 12 }); // history.pushState #1
```

### 实现

```typescript
// 批量更新的内部实现
function useBatchUpdater() {
  const pendingUpdates = useRef(new Map<string, any>());
  const flushScheduled = useRef(false);

  const scheduleFlush = () => {
    if (flushScheduled.current) return;
    flushScheduled.current = true;

    // 在下一个事件循环中批量更新
    queueMicrotask(() => {
      const updates = new Map(pendingUpdates.current);
      pendingUpdates.current.clear();
      flushScheduled.current = false;

      // 一次性更新所有参数
      const newParams = new URLSearchParams(window.location.search);
      for (const [key, value] of updates) {
        if (value === null) {
          newParams.delete(key);
        } else {
          newParams.set(key, value);
        }
      }
      window.history.pushState(null, '', `?${newParams}`);
    });
  };

  return { scheduleFlush, pendingUpdates };
}
```

## 服务器端渲染 (SSR)

### 数据脱水/水合

nuqs 支持 SSR，通过适配器实现状态的序列化与反序列化。

### 1. 服务端定义 Parser

```typescript
// nuqs/server 导出
import { createSearchParamsCache, parseAsString } from 'nuqs/server';

export const voicesSearchParams = {
  query: parseAsString.withDefault(''),
};

// 创建缓存对象
export const voicesSearchParamsCache =
  createSearchParamsCache(voicesSearchParams);
```

### 2. 服务端预取

```typescript
// page.tsx (Server Component)
import { SearchParams } from 'nuqs/server';

export default async function Page({
  searchParams,
}: {
  searchParams: Promise<SearchParams>;
}) {
  // 解析 URL 参数
  const { query } = await voicesSearchParamsCache.parse(searchParams);

  // 使用参数查询数据
  const data = await db.voices.findMany({
    where: { name: { contains: query } }
  });

  // 传递给客户端
  return <VoiceList data={data} />;
}
```

### 3. createSearchParamsCache 内部实现

`createSearchParamsCache` 是专门给服务端使用的工具，它将多个 Parser 组合成一个缓存对象：

```typescript
function createSearchParamsCache<T extends Record<string, Parser>>(
  parsers: T
) {
  return {
    // 解析 SearchParams 对象
    parse: async (searchParams: SearchParams) => {
      const result = {};
      for (const [key, parser] of Object.entries(parsers)) {
        const value = searchParams.get(key);
        result[key] = parser.parse(value);
      }
      return result;
    },

    // 序列化回 URL（用于构建链接）
    serialize: (values: Partial<Values<T>>) => {
      const params = new URLSearchParams();
      for (const [key, value] of Object.entries(values)) {
        const serialized = parsers[key].serialize(value);
        if (serialized) {
          params.set(key, serialized);
        }
      }
      return params.toString();
    }
  };
}
```

## 性能优化

### 1. 防抖 (Debounce)

对于输入类组件，需要防抖避免频繁更新 URL：

```typescript
// 项目中的实际用法
import { useDebouncedCallback } from 'use-debounce';

const debouncedSetQuery = useDebouncedCallback(
  (value: string) => setQuery(value),
  300
);

<input
  value={localQuery}
  onChange={e => {
    setLocalQuery(e.target.value); // 快速响应用户输入
    debouncedSetQuery(e.target.value); // 防抖更新 URL
  }}
/>
```

### 2. 时间安全 (Time Safety)

nuqs 确保：
- UI 更新立即响应（React 状态优先）
- URL 更新异步执行
- 两者解耦，避免阻塞

```typescript
// setter 返回 Promise，可以在 URL 更新后执行操作
const handleSet = async () => {
  setCount(5);
  // UI 已经更新，但 URL 可能还没更新
  await setCount(5); // 等待 URL 更新完成
  // 现在 URL 也更新了
};
```

### 3. Shallow Routing

默认使用浅层路由，只更新搜索参数不触发组件重挂载：

```tsx
// 默认 shallow: true
useQueryState('page', parseAsInteger.withDefault(1));

// 禁用浅层路由，会触发完整导航
useQueryState('page', {
  ...parseAsInteger.withDefault(1),
  shallow: false
});
```

## 历史管理

### 1. history 选项

```typescript
// 替换当前历史记录（默认）
useQueryState('query', {
  defaultValue: '',
  history: 'replace'
});

// 添加新历史记录
useQueryState('query', {
  defaultValue: '',
  history: 'push'
});
```

### 2. 浏览器历史导航

当用户点击浏览器后退/前进按钮时，nuqs 会：
1. 检测 URL 变化
2. 重新解析参数
3. 更新 React 状态
4. 触发组件重新渲染

## 类型安全实现

### 1. TypeScript 泛型

```typescript
// parser.type: number (不再是 number | null)

// useQueryState 返回值类型自动推断
const [count, setCount] = useQueryState('count', parser);
// count: number (非 null!)
```

## 总结

nuqs 的核心原理：

1. **Hook 包装**：将 useState 包装为 useQueryState
2. **Parser 解析**：分离解析/序列化逻辑到独立的 Parser 对象
3. **Adapter 抽象**：通过适配器屏蔽框架差异
4. **批量更新**：合并多次更新为一次 URL 修改
5. **类型推断**：利用 TypeScript 泛型实现端到端类型安全

这种设计让开发者可以像使用 useState 一样简单，同时获得 URL 状态管理的全部好处。

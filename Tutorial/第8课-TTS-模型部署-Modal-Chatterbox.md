## 一、Modal 平台简介

### 什么是 Modal？

Modal 是一个 Serverless GPU 云平台，允许你使用 Python 代码定义函数，然后在云端 GPU 基础设施上运行它们，无需管理服务器。

```
┌──────────────────────────────────────────────┐
│                 开发流程                       │
├──────────────────────────────────────────────┤
│                                              │
│  本地写 Python 代码  →  modal deploy  →  云端运行 │
│                                              │
│  ┌──────────┐    ┌──────────────┐  ┌────────┐│
│  │chatterbox│ →  │modal deploy  │→ │GPU API ││
│  │_tts.py   │    │              │  │Endpoint││
│  └──────────┘    └──────────────┘  └────────┘│
└──────────────────────────────────────────────┘
```

### Modal vs 其他 GPU 平台

| 平台 | 计费方式 | 启动速度 | 适用场景 |
|------|----------|----------|----------|
| **Modal** | 按秒计费，无请求不收费 | 秒级冷启动 | 间歇性推理任务 |
| **GPU 云服务器** | 包月/包年 | 即时 | 持续高负载 |
| **Replicate** | 按调用次数 | 较快 | 简单模型部署 |
| **AWS SageMaker** | 实例+推理 | 分钟级 | 企业级 ML Pipeline |

### 为什么项目选择 Modal？

1. **按秒计费**：TTS 生成是间歇性请求，Modal 无请求时不收费
2. **GPU 自动伸缩**：无需管理实例，冷启动后自动分配 GPU
3. **Python 原生**：模型推理代码直接用 Python 编写
4. **存储挂载**：可以挂载 R2 存储桶作为语音样本库

## 二、Chatterbox TTS 模型

### 什么是 Chatterbox？

Chatterbox 是一个零样本语音克隆 TTS 模型，特点是：
- **零样本克隆**：只需一段 10 秒以上的参考音频，就能模仿该声音
- **高质量输出**：生成自然的语音，支持多种语言
- **参数可调**：支持 temperature、topP、topK 等参数控制风格

### 模型工作原理

```
┌─────────────────────────────────────────────────────────┐
│                  Chatterbox TTS 流程                     │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  文本输入          语音样本           输出               │
│  ┌──────┐         ┌──────┐         ┌──────┐            │
│  │"Hello│         │参考  │         │生成的 │            │
│  │world"│    +    │音频  │    →    │音频  │            │
│  │      │         │10s+ │         │.wav  │            │
│  └──────┘         └──────┘         └──────┘            │
│                                                         │
│  模型内部流程：                                          │
│  1. 文本编码 → 文本特征向量                              │
│  2. 语音编码 → 音色特征向量（零样本克隆）                 │
│  3. 声学模型 → 生成声学特征                              │
│  4. 声码器 → 波形输出                                    │
└─────────────────────────────────────────────────────────┘
```

### 零样本语音克隆 vs 传统语音克隆

| 方式 | 训练数据 | 训练时间 | 克隆速度 |
|------|----------|----------|----------|
| **传统方式** | 数小时音频 | 数小时 | 实时 |
| **零样本（Chatterbox）** | 10-30 秒音频 | 无需训练 | 实时 |

  },

### openapi-fetch 是什么？

`openapi-fetch` 是一个类型安全的 HTTP 客户端库，它：
1. 从 OpenAPI 规范自动生成类型定义
2. 提供完整的 TypeScript 类型推导
3. 支持路径、请求体、响应体的自动类型检查

```
OpenAPI Spec (JSON)
         ↓
  openapi-typescript (scripts/sync-api.ts)
         ↓
  TypeScript 类型定义 (chatterbox-api.d.ts)
         ↓
  openapi-fetch 客户端 (chatterbox-client.ts)
         ↓
  类型安全的 API 调用
```

### 为什么使用 openapi-fetch？

| 方式 | 类型安全 | 自动补全 | 维护成本 |
|------|----------|----------|----------|
| **手写 fetch** | 需手动定义 | 无 | 高（API 变化需同步修改）|
| **openapi-fetch** | 自动生成 | 完整 | 低（只需同步 OpenAPI spec）|
| **Axios + 类型** | 需手动定义 | 部分 | 中 |

## 四、API 类型同步

### 同步脚本

**`scripts/sync-api.ts`**：

```typescript
import fs from "node:fs";
import path from "node:path";
import { fileURLToPath } from "node:url";
import "dotenv/config";
import openapiTS, { astToString } from "openapi-typescript";

const __dirname = path.dirname(fileURLToPath(import.meta.url));
const OUTPUT_PATH = path.resolve(__dirname, "../src/types/chatterbox-api.d.ts");

async function main() {
  const apiUrl = process.env.CHATTERBOX_API_URL;
  if (!apiUrl) {
    console.error("Error: CHATTERBOX_API_URL environment variable is required");
    process.exit(1);
  }

  // 1. 获取 OpenAPI spec
  const openApiUrl = `${apiUrl}/openapi.json`;
  console.log(`Fetching OpenAPI spec from: ${openApiUrl}`);

  // 2. 生成 TypeScript AST
  const ast = await openapiTS(new URL(openApiUrl));
  const contents = astToString(ast);

  // 3. 写入文件
  const header = `/**
* This file is auto-generated by scripts/sync-api.ts
* Do not edit manually. Run \`npm run sync-api\` to regenerate.
*
* Generated from: ${openApiUrl}
* Generated at: ${new Date().toISOString()}
*/`;

  fs.writeFileSync(OUTPUT_PATH, header + contents);
  console.log(`Types written to: ${OUTPUT_PATH}`);
}

main().catch((err) => {
  console.error("Failed to sync API types:", err);
  process.exit(1);
});
```

### 使用方式

```bash

# 方式 2: 临时指定 URL
CHATTERBOX_API_URL=https://your-api-url npm run sync-api
```

### 参数作用详解

#### 1. Temperature（温度）

```
范围: 0 - 2
默认: 0.8

Temperature 控制输出的随机性和创造性：

低温 (0.1-0.5)：               高温 (1.2-2.0)：
  输出稳定、可预测               输出多样、富有表现力
  适合：新闻播报、教程           适合：讲故事、表演
  "Consistent"                 "Expressive"
```

#### 2. Top P（核采样）

```
范围: 0 - 1
默认: 0.95

Top P 控制采样时的词汇范围：

低 Top P (0.5-0.8)：            高 Top P (0.9-1.0)：
  只从高概率词中选择             考虑更多低概率词
  输出稳定、保守                 输出多样、有创意
  "Stable"                     "Dynamic"
```

#### 3. Top K

```
范围: 1 - 10000
默认: 1000

Top K 限制采样时考虑的候选词数量：

低 Top K (1-100)：              高 Top K (1000-10000)：
  只考虑最可能的 K 个词          考虑更多候选词
  输出一致、单调                 输出丰富、多变
  "Subtle"                     "Dramatic"
```

#### 4. Repetition Penalty（重复惩罚）

```
范围: 1 - 2
默认: 1.2

防止模型重复生成相同内容：

低惩罚 (1.0-1.1)：              高惩罚 (1.5-2.0)：
  允许一定重复                   强烈避免重复
  适合：韵律、强调               适合：自然对话
  "Rhythmic"                   "Varied"
```

### 参数预设推荐

| 场景 | Temperature | Top P | Top K | Repetition Penalty |
|------|-------------|-------|-------|--------------------|
| **新闻播报** | 0.5 | 0.9 | 500 | 1.1 |
| **日常对话** | 0.8 | 0.95 | 1000 | 1.2 |
| **讲故事** | 1.2 | 0.98 | 2000 | 1.3 |
| **诗歌朗诵** | 1.5 | 1.0 | 3000 | 1.5 |

### 表单提交处理

```typescript
export function TextToSpeechForm({ children, defaultValues }) {
  const trpc = useTRPC();
  const router = useRouter();
  const createMutation = useMutation(
    trpc.generations.create.mutationOptions({}),
  );
  const { checkout } = useCheckout();

  const form = useAppForm({
    ...ttsFormOptions,
    defaultValues: defaultValues ?? defaultTTSValues,
    validators: { onSubmit: ttsFormSchema },
    onSubmit: async ({ value }) => {
      try {
        const data = await createMutation.mutateAsync({
          text: value.text.trim(),
          voiceId: value.voiceId,
          temperature: value.temperature,
          topP: value.topP,
          topK: value.topK,
          repetitionPenalty: value.repetitionPenalty,
        });

        toast.success("Audio generated successfully!");
        router.push(`/text-to-speech/${data.id}`);
      } catch (error) {
        const message = error instanceof Error ? error.message : "Failed to generate audio";
        if (message === "SUBSCRIPTION_REQUIRED") {
          toast.error("Subscription required", {
            action: { label: "Subscribe", onClick: () => checkout() },
          });
        } else {
          toast.error(message);
        }
      }
    },
  });

  return <form.AppForm>{children}</form.AppForm>;
}
```

## 七、完整 TTS 生成链路

### 端到端流程

```
用户输入文本 → 选择语音 → 调节参数 → 点击「Generate speech」
                                                    ↓
┌─────────────────────────────────────────────────────────┐
│  Step 1: 前端表单提交                                    │
│  ┌──────────────────────────────────────────────────┐   │
│  │ TextToSpeechForm.onSubmit                        │   │
│  │ → validate (Zod schema)                          │   │
│  │ → createMutation.mutateAsync()                   │   │
│  └──────────────────────┬───────────────────────────┘   │
└─────────────────────────┼───────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Step 2: tRPC 中间层                                     │
│  ┌──────────────────────────────────────────────────┐   │
│  │ POST /api/trpc/generations.create                │   │
│  │ → SentryMiddleware (日志记录)                     │   │
│  │ → orgProcedure (认证 + orgId 注入)               │   │
│  │ → Zod validation (input 验证)                   │   │
│  └──────────────────────┬───────────────────────────┘   │
└─────────────────────────┼───────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Step 3: tRPC Mutation 处理                              │
│  ┌──────────────────────────────────────────────────┐   │
│  │ 1. 检查订阅状态 (Polar)                          │   │
│  │ 2. 查询语音信息 (Prisma)                         │   │
│  │ 3. 调用 Chatterbox API                           │   │
│  │ 4. 接收音频 ArrayBuffer                          │   │
│  └──────────────────────┬───────────────────────────┘   │
└─────────────────────────┼───────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Step 4: 存储音频                                        │
│  ┌──────────────────────────────────────────────────┐   │
│  │ 1. 创建 Generation 记录 (DB)                     │   │
│  │ 2. 定义 R2 Key: generations/orgs/{orgId}/{id}    │   │
│  │ 3. uploadAudio() → R2                            │   │
│  │ 4. 更新 r2ObjectKey (DB)                         │   │
│  └──────────────────────┬───────────────────────────┘   │
└─────────────────────────┼───────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Step 5: 返回结果                                        │
│  ┌──────────────────────────────────────────────────┐   │
│  │ return { id: generationId }                      │   │
│  │ → toast.success("Audio generated!")              │   │
│  │ → router.push(`/text-to-speech/${data.id}`)      │   │
│  └──────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

### 核心调用代码

**`src/trpc/routers/generations.ts`**（Chatterbox 调用部分）：

```typescript
import { chatterbox } from "@/lib/chatterbox-client";

// 调用 Chatterbox TTS API
const { data, error } = await chatterbox.POST("/generate", {
  body: {
    prompt: input.text,                      // 要合成的文本
    voice_key: voice.r2ObjectKey,            // 语音样本的 R2 Key
    temperature: input.temperature,          // 温度参数
    top_p: input.topP,                       // Top P 采样
    top_k: input.topK,                       // Top K 采样
    repetition_penalty: input.repetitionPenalty,
    norm_loudness: true,                     // 响度归一化
  },
  parseAs: "arrayBuffer",                    // 以二进制接收音频
});

if (error) {
  throw new TRPCError({
    code: "INTERNAL_SERVER_ERROR",
    message: "Failed to generate audio",
  });
}

// 将 ArrayBuffer 转为 Buffer 用于上传
const buffer = Buffer.from(data);
```

### 响应处理

```
Chatterbox API 响应:
  Content-Type: audio/wav
  Body: [binary audio data]

处理方式:
  parseAs: "arrayBuffer" → 接收为 ArrayBuffer
  Buffer.from(data) → 转为 Node.js Buffer
  uploadAudio({ buffer, key }) → 上传到 R2
```

## 九、Modal 部署（概念）

### 典型的 Modal 部署脚本

虽然项目源码中没有包含 Python 部署文件（Modal 服务是独立部署的），但一个典型的 Chatterbox TTS Modal 部署大致如下：

```python
import modal
from modal import Mount

app = modal.App("chatterbox-tts")

# GPU 配置
image = modal.Image.debian_slim().pip_install(
    "torch", "torchaudio", "transformers", "chatterbox-tts"
)

@app.function(
    gpu="A10G",
    timeout=300,  # 5 分钟超时
    image=image,
)
@modal.web_endpoint(method="POST")
def generate(request: dict):
    """TTS 生成端点"""
    from chatterbox.tts import ChatterboxTTS

    model = ChatterboxTTS.from_pretrained()

    # 从 R2 加载语音样本
    voice_key = request["voice_key"]
    reference_audio = load_from_r2(voice_key)

    # 生成音频
    audio = model.generate(
        text=request["prompt"],
        prompt_audio=reference_audio,
        temperature=request.get("temperature", 0.8),
        top_p=request.get("top_p", 0.95),
        top_k=request.get("top_k", 1000),
        repetition_penalty=request.get("repetition_penalty", 1.2),
    )

    return modal.Audio(audio, sample_rate=24000)
```

## 十、错误处理与重试

### 调用链中的错误处理

```
前端 (TextToSpeechForm)
  catch → toast.error(message) → 订阅检查/通用错误
    ↑
tRPC (generations.create)
  throw TRPCError → SUBSCRIPTION_REQUIRED / INTERNAL_SERVER_ERROR
    ↑
Chatterbox API
  error → 网络错误 / 模型错误 / 超时
```

### 完整的错误处理流程

**`src/trpc/routers/generations.ts`**：

```typescript
// 1. 订阅检查
try {
  const customerState = await polar.customers.getStateExternal({
    externalId: ctx.orgId,
  });
  if ((customerState.activeSubscriptions ?? []).length === 0) {
    throw new TRPCError({
      code: "FORBIDDEN",
      message: "SUBSCRIPTION_REQUIRED",
    });
  }
} catch (err) {
  if (err instanceof TRPCError) throw err;
  throw new TRPCError({
    code: "FORBIDDEN",
    message: "SUBSCRIPTION_REQUIRED",
  });
}

// 2. 语音验证
if (!voice) {
  throw new TRPCError({ code: "NOT_FOUND", message: "Voice not found" });
}

if (!voice.r2ObjectKey) {
  throw new TRPCError({
    code: "PRECONDITION_FAILED",
    message: "Voice audio not available",
  });
}

// 3. Chatterbox 调用
const { data, error } = await chatterbox.POST("/generate", {
  body: { /* ... */ },
  parseAs: "arrayBuffer",
});

if (error) {
  throw new TRPCError({
    code: "INTERNAL_SERVER_ERROR",
    message: "Failed to generate audio",
  });
}

// 4. 存储阶段错误处理
try {
  await uploadAudio({ buffer, key: r2ObjectKey });
} catch {
  // 回滚数据库记录
  await prisma.generation.delete({ where: { id: generationId } }).catch(() => {});
  throw new TRPCError({
    code: "INTERNAL_SERVER_ERROR",
    message: "Failed to store generated audio",
  });
}
```

#### 步骤 3：创建客户端表单组件

创建 `src/app/(dashboard)/tts-practice/tts-practice-form.tsx`：

```typescript
"use client";

import { useState } from "react";
import { useRouter } from "next/navigation";
import { useMutation, useSuspenseQuery } from "@tanstack/react-query";
import { useTRPC } from "@/trpc/client";
import { toast } from "sonner";
import { sliders } from "@/features/text-to-speech/data/sliders";
import { TEXT_MAX_LENGTH } from "@/features/text-to-speech/data/constants";

export function TTSPracticeForm() {
  const trpc = useTRPC();
  const router = useRouter();
  const [text, setText] = useState("");
  const [voiceId, setVoiceId] = useState("");
  const [params, setParams] = useState({
    temperature: 0.8,
    topP: 0.95,
    topK: 1000,
    repetitionPenalty: 1.2,
  });

  const { data: voices } = useSuspenseQuery(
    trpc.voices.getAll.queryOptions()
  );
  const allVoices = [...voices.custom, ...voices.system];

  // 初始化默认语音
  if (!voiceId && allVoices.length > 0) {
    setVoiceId(allVoices[0].id);
  }

  const createMutation = useMutation(
    trpc.generations.create.mutationOptions({})
  );

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    if (!text.trim() || !voiceId) return;

    try {
      const data = await createMutation.mutateAsync({
        text: text.trim(),
        voiceId,
        ...params,
      });
      toast.success("生成成功！");
      router.push(`/text-to-speech/${data.id}`);
    } catch (error) {
      toast.error(
        error instanceof Error ? error.message : "生成失败"
      );
    }
  };

  return (
    <form onSubmit={handleSubmit} className="space-y-6">
      {/* 文本输入 */}
      <div>
        <label className="block text-sm font-medium mb-2">
          输入文本（最多 {TEXT_MAX_LENGTH} 字符）
        </label>
        <textarea
          value={text}
          onChange={(e) => setText(e.target.value)}
          maxLength={TEXT_MAX_LENGTH}
          className="w-full p-3 border rounded-lg h-32 resize-none"
          placeholder="请输入要转换为语音的文本..."
        />
        <p className="text-xs text-muted-foreground mt-1">
          {text.length} / {TEXT_MAX_LENGTH} 字符
        </p>
      </div>

      {/* 语音选择 */}
      <div>
        <label className="block text-sm font-medium mb-2">选择语音</label>
        <select
          value={voiceId}
          onChange={(e) => setVoiceId(e.target.value)}
          className="w-full p-3 border rounded-lg"
        >
          {allVoices.map((v) => (
            <option key={v.id} value={v.id}>
              {v.name} ({v.variant === "SYSTEM" ? "系统" : "自定义"})
            </option>
          ))}
        </select>
      </div>

      {/* 参数调节 */}
      <div className="space-y-4">
        <h3 className="text-sm font-medium">生成参数</h3>
        {sliders.map((slider) => (
          <div key={slider.id} className="space-y-1">
            <div className="flex justify-between text-sm">
              <span>{slider.label}</span>
              <span className="text-muted-foreground">
                {params[slider.id]}
              </span>
            </div>
            <input
              type="range"
              min={slider.min}
              max={slider.max}
              step={slider.step}
              value={params[slider.id]}
              onChange={(e) =>
                setParams((prev) => ({
                  ...prev,
                  [slider.id]: parseFloat(e.target.value),
                }))
              }
              className="w-full"
            />
            <div className="flex justify-between text-xs text-muted-foreground">
              <span>{slider.leftLabel}</span>
              <span>{slider.rightLabel}</span>
            </div>
          </div>
        ))}
      </div>

      {/* 提交按钮 */}
      <button
        type="submit"
        disabled={createMutation.isPending || !text.trim() || !voiceId}
        className="w-full py-3 bg-primary text-primary-foreground rounded-lg font-medium hover:opacity-90 disabled:opacity-50"
      >
        {createMutation.isPending ? "生成中..." : "生成语音"}
      </button>
    </form>
  );
}
```

## 常见问题 FAQ

### Q: Chatterbox API 返回超时怎么办？

A: 可能原因：
1. GPU 冷启动需要时间（首次请求可能 30-60 秒）
2. 文本过长导致处理时间增加
3. Modal 实例未运行

解决方案：
- 在 Modal Dashboard 中设置最小实例数为 1，避免冷启动
- 增加 tRPC 超时设置

### Q: 如何实现流式生成？

A: 当前项目是同步等待整个音频生成完成后再返回。如果需要流式：
```typescript
// 使用 SSE (Server-Sent Events)
const response = await fetch("/api/generate-stream", {
  method: "POST",
  body: JSON.stringify({ text, voiceId }),
});
const reader = response.body?.getReader();
// 逐块读取音频数据
```

### Q: 为什么使用 openapi-fetch 而不是直接 fetch？

A: openapi-fetch 提供：
- 端到端类型安全（参数和响应体自动推导）
- API 变化时 TypeScript 编译器自动报错
- 无需手写类型定义，减少维护成本

### Q: 如何更新 API 类型？

A: 当 Modal 端点更新后：
```bash
# 重新获取 OpenAPI spec 并生成类型
npm run sync-api

# 检查生成的类型文件
git diff src/types/chatterbox-api.d.ts
```

### Q: TEXT_MAX_LENGTH 5000 字符的限制是如何确定的？

A: 基于以下考虑：
1. Chatterbox 模型的上下文窗口限制
2. 生成时长控制（避免过长文本导致超时）
3. 成本考虑（字符数与费用挂钩）

## 6.2 调试工具与方法

### 6.2.1 TypeScript 类型推断

tRPC 最大的优势之一是端到端的类型安全。让我们看看如何使用这个特性：

**在编辑器中查看类型**

```typescript
// 鼠标悬停在 trpc 上，可以看到返回类型
const trpc = useTRPC();

// query 返回的数据类型自动推断
const { data } = useSuspenseQuery(trpc.voices.getAll.queryOptions());

// data 的类型自动显示：
// { custom: Voice[], system: Voice[] }
```

**跳转到类型定义**

在 VS Code 中：
1. 右键点击 `AppRouter` 
2. 选择 "Go to Type Definition"
3. 可以看到完整的 Router 类型定义

### 6.2.2 API 调用流程追踪

当我们调用一个 tRPC API 时，流程是这样的：

```
前端调用                           服务端处理
─────────────────────────────    ─────────────────────
trpc.voices.getAll.query()       
        │                        
        │  POST /api/trpc/voices.getAll
        ├─────────────────────────►
        │                        │
        │                        ├── 验证输入 (Zod)
        │                        ├── 执行中间件
        │                        ├── 执行 Procedure
        │                        └── 返回结果
        ◄────────────────────────┤
        │                        
返回 data (类型已知)
```

**调试技巧：**

1. **查看网络请求**
   - 打开浏览器开发者工具 → Network
   - 找到 `/api/trpc/*` 的请求
   - 查看 Request 和 Response

2. **查看请求内容**
   ```json
   // Request
   {
     "json": {
       "id": "voices.getAll",
       "params": {}
     }
   }
   
   // Response  
   {
     "result": {
       "data": {
         "custom": [...],
         "system": [...]
       }
     }
   }
   ```

## 6.3 常见问题与解决方案

### 6.3.1 认证问题

**问题：UNAUTHORIZED 错误**

```typescript
// 错误信息
{
  "error": {
    "message": "Unauthorized",
    "code": "UNAUTHORIZED"
  }
}
```

**解决方案：**
1. 检查用户是否登录
2. 检查 session 是否过期
3. 检查 token 是否正确传递

```typescript
// 调试：打印 auth 信息
export const authProcedure = baseProcedure.use(async ({ next }) => {
  const authResult = await auth();
  console.log('Auth result:', authResult);
  
  const { userId } = authResult;
  if (!userId) {
    throw new TRPCError({ code: "UNAUTHORIZED" });
  }
  
  return next({ ctx: { userId } });
});
```

### 6.3.2 权限问题

**问题：FORBIDDEN 错误**

```typescript
// 错误信息
{
  "error": {
    "message": "Organization required",
    "code": "FORBIDDEN"
  }
}
```

**解决方案：**
1. 检查用户是否属于组织
2. 检查组织 ID 是否正确

### 6.3.3 类型不匹配问题

**问题：前端类型与后端不一致**

**症状：**
- 编辑器报错：Type 'X' is not assignable to type 'Y'
- 数据结构与预期不符

**解决方案：**

1. **确保 TypeScript 配置正确**
   ```json
   // tsconfig.json
   {
     "compilerOptions": {
       "moduleResolution": "bundler",
       "paths": {
         "@/*": ["./src/*"]
       }
     }
   }
   ```

2. **检查导入路径**
   ```typescript
   // ❌ 错误
   import type { AppRouter } from '../routers/_app';
   
   // ✅ 正确
   import type { AppRouter } from '@/trpc/routers/_app';
   ```

3. **重新运行 TypeScript**
   ```bash
   npx tsc --noEmit
   ```

### 6.3.4 数据转换问题

**问题：superjson 序列化失败**

**症状：**
- 数据为 null 或 undefined
- 日期变成字符串
- BigInt 无法使用

**解决方案：**

1. **检查服务端和客户端配置**
   ```typescript
   // 服务端
   const t = initTRPC.create({
     transformer: superjson,
   });
   
   // 客户端
   httpBatchLink({
     transformer: superjson,  // 必须配置
     url: getUrl(),
   })
   ```

2. **检查 query-client 配置**
   ```typescript
   // 必须配置 serialize/deserialize
   dehydrate: {
     serializeData: superjson.serialize,
   },
   hydrate: {
     deserializeData: superjson.deserialize,
   }
   ```

### 6.3.5 缓存问题

**问题：数据不更新**

**症状：**
- 更新数据后，页面显示旧数据
- 删除数据后，仍然显示

**解决方案：**

1. **手动刷新数据**
   ```typescript
   const utils = trpc.useUtils();
   
   // 在 mutation 成功后刷新
   const deleteMutation = useMutation(
     trpc.voices.delete.mutationOptions({
       onSuccess: () => {
         utils.voices.getAll.invalidate();  // 刷新列表
       },
     })
   );
   ```

2. **调整 staleTime**
   ```typescript
   // 缩短 staleTime
   const { data } = useSuspenseQuery(
     trpc.voices.getAll.queryOptions({
       staleTime: 5000,  // 5秒
     })
   );
   ```

3. **手动设置数据**
   ```typescript
   utils.voices.getAll.setData(
     { query: '' },
     (oldData) => {
       // 过滤掉被删除的项
       return oldData?.filter(v => v.id !== deletedId);
     }
   );
   ```

### 6.3.6 SSR 水合问题

**问题：Hydration mismatch**

**症状：**
- 控制台警告：Hydration failed
- 页面闪烁

**解决方案：**

1. **检查数据是否正确序列化**
   ```typescript
   // 确保服务端和客户端使用相同的 query client 配置
   ```

2. **使用 HydrationBoundary**
   ```typescript
   import { HydrateClient } from "@/trpc/server";
   
   // 在页面中
   <HydrateClient>
     <YourComponent />
   </HydrateClient>
   ```

3. **避免在服务端预取带有随机数据的查询**
   ```typescript
   // ❌ 避免
   prefetch(trpc.random.get.queryOptions());
   
   // ✅ 推荐
   prefetch(trpc.voices.getAll.queryOptions());
   ```

## 6.4 性能优化

### 6.4.1 请求批量处理

tRPC 默认使用 `httpBatchLink`，它会自动批量处理请求：

```
请求1: trpc.voices.getAll
请求2: trpc.generations.getAll
请求3: trpc.billing.getStatus

        ↓ httpBatchLink 自动合并

POST /api/trpc/voices.getAll,generations.getAll,billing.getStatus
```

**优化建议：**
- 保持默认配置即可
- 批量处理对大多数场景足够高效

### 6.4.2 数据缓存策略

**staleTime 设置**

```typescript
// query-client.ts
defaultOptions: {
  queries: {
    staleTime: 30 * 1000,  // 30秒
  },
}
```

**不同场景的推荐配置：**

```typescript
// 频繁更新的数据（如实时数据）
staleTime: 0

// 用户数据（如列表）
staleTime: 30 * 1000

// 静态数据（如配置）
staleTime: Infinity
```

### 6.4.3 避免不必要的数据请求

**使用 queryKey 精确控制缓存：**

```typescript
// 基础查询
trpc.voices.getAll.queryOptions()

// 带参数的查询 - 独立的缓存
trpc.voices.getAll.queryOptions({ query: 'search' })
trpc.voices.getAll.queryOptions({ query: 'another' })

// 不带参数 - 共享缓存
trpc.voices.getAll.queryOptions()
```

### 6.4.4 预取优化

**在页面中合理预取：**

```typescript
// page.tsx
export default async function VoicesPage({
  searchParams,
}) {
  const { query } = await searchParams;
  
  // 只预取需要的数据
  prefetch(trpc.voices.getAll.queryOptions({ query }));
  
  return (
    <HydrateClient>
      <VoicesView />
    </HydrateClient>
  );
}
```

### 6.4.5 使用 useSuspenseQuery

```typescript
// ✅ 推荐：useSuspenseQuery
// - 不会触发 loading 状态
// - 配合 Suspense 使用
// - 适合 SSR
const { data } = useSuspenseQuery(trpc.voices.getAll.queryOptions());

// ⚠️ useQuery：可能导致 loading 状态
// - 需要处理 loading/error 状态
// - 适合客户端导航后的动态加载
const { data, isLoading } = useQuery(
  trpc.voices.getAll.queryOptions()
);
```

### 6.4.6 优化 Mutation 性能

**使用 optimistic update：**

```typescript
const deleteMutation = useMutation(
  trpc.voices.delete.mutationOptions({
    onMutate: async (newDelete) => {
      // 取消所有正在进行的查询
      await utils.voices.getAll.cancel();
      
      // 获取当前数据
      const previousData = utils.voices.getAll.getData();
      
      // 乐观更新 - 立即移除
      utils.voices.getAll.setData(
        { query: '' },
        (old) => old?.filter(v => v.id !== newDelete.id)
      );
      
      // 返回之前的数据，用于回滚
      return { previousData };
    },
    onError: (err, newDelete, context) => {
      // 回滚到之前的数据
      utils.voices.getAll.setData(
        { query: '' },
        context?.previousData
      );
    },
    onSettled: () => {
      // 同步服务器数据
      utils.voices.getAll.invalidate();
    },
  })
);
```

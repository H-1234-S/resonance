## 7.2 练习一：添加新的 API 端点

### 7.2.1 任务描述

在 `voices` Router 中添加一个新的端点，用于获取声音的详情信息。

### 7.2.2 需求分析

1. 输入：声音 ID
2. 输出：声音的完整信息（包括数据库中未暴露的字段）
3. 权限：需要组织权限

### 7.2.3 实现步骤

**第一步：在 voices.ts 中添加 getById**

```typescript
// src/trpc/routers/voices.ts

getById: orgProcedure
  .input(z.object({ id: z.string() }))
  .query(async ({ ctx, input }) => {
    // 查询声音，需要验证：
    // 1. 声音存在
    // 2. 是系统声音 或 属于当前组织
    const voice = await prisma.voice.findFirst({
      where: {
        id: input.id,
        OR: [
          { variant: "SYSTEM" },
          { variant: "CUSTOM", orgId: ctx.orgId },
        ],
      },
    });

    if (!voice) {
      throw new TRPCError({
        code: "NOT_FOUND",
        message: "Voice not found",
      });
    }

    return voice;
  }),
```

**第二步：在前端组件中使用**


### 7.2.4 验证要点

## 7.3 练习二：创建新的 Router

### 7.3.1 任务描述

创建一个用户偏好设置的 Router，用于保存和获取用户的偏好配置。

### 7.3.2 需求分析

需要实现：
1. `getPreferences` - 获取用户偏好
2. `updatePreferences` - 更新用户偏好
3. 数据存储在数据库中

### 7.3.3 实现步骤

**第一步：创建 Router 文件**

```typescript
// src/trpc/routers/preferences.ts

import { z } from "zod";
import { TRPCError } from "@trpc/server";
import { prisma } from "@/lib/db";
import { createTRPCRouter, orgProcedure } from "../init";

// 偏好设置 schema
const preferencesSchema = z.object({
  theme: z.enum(["light", "dark", "system"]).default("system"),
  language: z.string().default("en"),
  notifications: z.object({
    email: z.boolean().default(true),
    push: z.boolean().default(false),
  }).optional(),
});

export const preferencesRouter = createTRPCRouter({
  get: orgProcedure.query(async ({ ctx }) => {
    const preferences = await prisma.preferences.findUnique({
      where: { orgId: ctx.orgId },
    });

    // 如果不存在，返回默认值
    if (!preferences) {
      return {
        theme: "system",
        language: "en",
        notifications: {
          email: true,
          push: false,
        },
      };
    }

    return preferences;
  }),

  update: orgProcedure
    .input(preferencesSchema)
    .mutation(async ({ ctx, input }) => {
      const preferences = await prisma.preferences.upsert({
        where: { orgId: ctx.orgId },
        create: {
          orgId: ctx.orgId,
          ...input,
        },
        update: input,
      });

      return preferences;
    }),
});
```

**第三步：在前端使用**

```typescript
// src/components/preferences-form.tsx

import { useTRPC } from "@/trpc/client";
import { useMutation, useSuspenseQuery } from "@tanstack/react-query";

export function PreferencesForm() {
  const trpc = useTRPC();
  
  // 获取偏好设置
  const { data } = useSuspenseQuery(
    trpc.preferences.get.queryOptions()
  );

  // 更新偏好设置
  const updateMutation = useMutation(
    trpc.preferences.update.mutationOptions({
      onSuccess: () => {
        console.log('Preferences updated');
      },
    })
  );

  const handleThemeChange = (theme: "light" | "dark" | "system") => {
    updateMutation.mutate({
      ...data,
      theme,
    });
  };

  return (
    <div>
      <select 
        value={data.theme} 
        onChange={(e) => handleThemeChange(e.target.value as any)}
      >
        <option value="light">Light</option>
        <option value="dark">Dark</option>
        <option value="system">System</option>
      </select>
    </div>
  );
}
```

## 7.4 练习三：实现 Optimistic Update

### 7.4.1 任务描述

为 `generations.create` 实现 optimistic update，让用户能够立即看到生成的音频，无需等待服务器响应。

### 7.4.2 需求分析

1. 创建时立即在列表中添加一个" pending"状态的项
2. 如果成功，替换为真实数据
3. 如果失败，回滚并显示错误

### 7.4.3 实现步骤

**第一步：创建一个展示生成记录的组件**

```typescript
// src/features/text-to-speech/components/generations-list.tsx

import { useTRPC } from "@/trpc/client";
import { useSuspenseQuery } from "@tanstack/react-query";

export function GenerationsList() {
  const trpc = useTRPC();
  
  const { data } = useSuspenseQuery(
    trpc.generations.getAll.queryOptions()
  );

  return (
    <div>
      {data.map((gen) => (
        <GenerationItem key={gen.id} generation={gen} />
      ))}
    </div>
  );
}

function GenerationItem({ generation }: { generation: any }) {
  return (
    <div className={generation.status === 'pending' ? 'opacity-50' : ''}>
      <p>{generation.text}</p>
      <p>Voice: {generation.voiceName}</p>
      <audio src={generation.audioUrl} controls />
    </div>
  );
}
```

**第二步：实现带 Optimistic Update 的 Creation**

```typescript
// src/features/text-to-speech/components/text-to-speech-form.tsx

import { useTRPC } from "@/trpc/client";
import { useMutation } from "@tanstack/react-query";

export function TextToSpeechForm() {
  const trpc = useTRPC();
  const utils = trpc.useUtils();

  const createMutation = useMutation(
    trpc.generations.create.mutationOptions({
      // 1. Optimistic Update - 在 mutation 开始前
      onMutate: async (newGeneration) => {
        // 取消所有正在进行的请求
        await utils.generations.getAll.cancel();
        await utils.generations.getById.cancel();

        // 获取之前的数据
        const previousGenerations = utils.generations.getAll.getData();

        // 乐观更新：立即添加一个新项
        const optimisticGeneration = {
          id: `temp-${Date.now()}`,
          text: newGeneration.text,
          voiceName: 'Loading...',
          voiceId: newGeneration.voiceId,
          status: 'pending' as const,
          audioUrl: '',
          createdAt: new Date(),
        };

        utils.generations.getAll.setData(
          undefined,
          (old) => old ? [optimisticGeneration, ...old] : [optimisticGeneration]
        );

        return { previousGenerations };
      },

      // 2. 错误处理 - 回滚
      onError: (err, newGeneration, context) => {
        // 回滚到之前的数据
        utils.generations.getAll.setData(
          undefined,
          context?.previousGenerations
        );
        
        // 显示错误
        console.error('Failed to create generation:', err.message);
      },

      // 3. 最终处理 - 无论成功或失败
      onSettled: () => {
        // 重新获取服务器数据确保一致
        utils.generations.getAll.invalidate();
      },
    })
  );

  const handleSubmit = (text: string, voiceId: string) => {
    createMutation.mutate({
      text,
      voiceId,
      temperature: 0.8,
    });
  };

  return (
    <form onSubmit={handleSubmit}>
      {/* 表单内容 */}
    </form>
  );
}
```

### 7.4.4 验证要点

- [ ] 在 mutation 开始前更新 UI
- [ ] 错误时回滚到之前的状态
- [ ] 最后与服务器同步
- [ ] 用户体验流畅

```typescript
// 工具函数：带重试的异步调用
async function withRetry<T>(
  fn: () => Promise<T>,
  maxRetries: number = 3,
  delay: number = 1000
): Promise<T> {
  let lastError: Error | undefined;
  
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn();
    } catch (error) {
      lastError = error as Error;
      if (i < maxRetries - 1) {
        await new Promise(resolve => setTimeout(resolve, delay));
      }
    }
  }
  
  throw lastError;
}

// 使用
createCheckout: orgProcedure.mutation(async ({ ctx }) => {
  const result = await withRetry(async () => {
    return await polar.checkouts.create({
      products: [env.POLAR_PRODUCT_ID],
      externalCustomerId: ctx.orgId,
    });
  });

  if (!result.url) {
    throw new TRPCError({
      code: "INTERNAL_SERVER_ERROR",
      message: "Failed to create checkout session",
    });
  }

  return { checkoutUrl: result.url };
}),
```

## 7.6 最佳实践总结

### 7.6.1 项目结构

```
src/trpc/
├── init.ts           # 初始化配置
├── query-client.ts  # React Query 配置
├── server.tsx       # SSR 支持
├── client.tsx       # 客户端 Provider
└── routers/
    ├── _app.ts     # 主路由
    ├── *.ts        # 子路由
```

### 7.6.2 命名规范

- Router: 使用功能名称（voices, billing）
- Procedure: 使用动词（get, create, update, delete）
- 文件名: 使用功能名称复数（voices.ts, billing.ts）

### 7.6.3 权限控制

- 使用中间件统一处理认证和权限
- 敏感操作使用 `authProcedure` 或 `orgProcedure`
- 在业务逻辑中再次验证权限（防御性编程）

### 7.6.4 错误处理

- 使用标准 TRPCError 错误码
- 在前端统一处理错误
- 使用 Sentry 追踪生产环境错误

### 7.6.5 性能优化

- 使用 `useSuspenseQuery` 替代 `useQuery`
- 合理设置 `staleTime`
- 使用 `httpBatchLink` 批量请求
- 使用 optimistic update 提升用户体验

## 5.2 Zod 高级验证

### 5.2.1 基础验证类型

在前面的课程中，我们已经见过一些基础验证。让我系统地整理一下：

```typescript
import { z } from "zod";

// 字符串验证
z.string()                    // 任意字符串
z.string().min(1)            // 最小长度
z.string().max(100)          // 最大长度
z.string().email()           // 邮箱格式
z.string().url()             // URL格式
z.string().uuid()           // UUID格式
z.string().trim()           // 去除首尾空格

// 数字验证
z.number()                   // 任意数字
z.number().min(0)           // 最小值
z.number().max(100)         // 最大值
z.number().int()            // 整数
z.number().positive()       // 正数
z.number().negative()       // 负数

// 布尔验证
z.boolean()                 // 布尔值

// 可选和 nullable
z.string().optional()       // 可选字符串（undefined）
z.string().nullable()      // 可为 null
z.string().nullish()      // 可选或 null

// 带默认值
z.string().default("")    // 默认值
z.number().default(0)     // 默认数字
```

### 5.2.2 复杂对象验证

```typescript
// 嵌套对象
const userSchema = z.object({
  id: z.string().uuid(),
  name: z.string().min(1).max(100),
  email: z.string().email(),
  profile: z.object({
    bio: z.string().optional(),
    avatar: z.string().url().optional(),
  }),
});

// 数组验证
const arraySchema = z.array(z.string());
const numberArraySchema = z.array(z.number().min(0));

// 数组长度限制
z.array(z.string()).min(1)           // 至少1个元素
z.array(z.string()).max(10)          // 最多10个元素
z.array(z.string()).length(5)        // 正好5个元素

// 数组元素验证
z.array(
  z.object({
    id: z.string(),
    name: z.string(),
  })
)
```

### 5.2.3 联合类型和枚举

```typescript
// 联合类型
const stringOrNumber = z.string().or(z.number());
const resultSchema = z.object({
  status: z.literal("success").or(z.literal("error")),
});

// 枚举
const roleSchema = z.enum(["admin", "user", "guest"]);

// 预处理验证
const idSchema = z.coerce.string().uuid();
```

### 5.2.5 自定义验证

Zod 允许我们创建自定义验证规则：

```typescript
// 自定义验证函数
const positiveInteger = z.number().int().positive();

// 使用 refine 添加自定义验证
const userSchema = z.object({
  age: z.number().min(0).max(150),
  email: z.string().email(),
}).refine(
  (data) => data.age >= 18 || data.email.endsWith('@minor.edu'),
  {
    message: "未成年用户需要使用教育邮箱",
    path: ["age"],
  }
);

// 使用 superRefine 进行复杂验证
const complexSchema = z.object({
  password: z.string().min(8),
  confirmPassword: z.string(),
}).superRefine((data, ctx) => {
  if (data.password !== data.confirmPassword) {
    ctx.addIssue({
      code: z.ZodIssueCode.custom,
      message: "密码不匹配",
      path: ["confirmPassword"],
    });
  }
});
```

### 5.2.6 验证错误处理

当验证失败时，Zod 会抛出错误：

```typescript
try {
  const data = schema.parse(input);
} catch (error) {
  if (error instanceof z.ZodError) {
    // 遍历所有错误
    error.errors.forEach((err) => {
      console.log(`字段 ${err.path.join('.')}: ${err.message}`);
    });
  }
}
```

## 5.3 Superjson 数据转换

### 5.3.1 为什么需要数据转换

JavaScript 有一些特殊的数据类型，JSON 无法直接处理：

```typescript
const data = {
  // Date 对象 → JSON 后变成字符串
  createdAt: new Date(),
  
  // BigInt → JSON 不支持
  bigNumber: BigInt(123456789),
  
  // undefined → JSON 会忽略
  undefinedValue: undefined,
  
  // Set → JSON 不支持
  tags: new Set(['a', 'b']),
  
  // Symbol → JSON 不支持
  symbol: Symbol('test'),
};
```

### 5.3.2 Superjson 的解决方案

```typescript
import superjson from "superjson";

// 序列化
const jsonString = superjson.stringify(data);
// {
//   "createdAt": "2024-01-01T00:00:00.000Z",
//   "bigNumber": {"__BigInt": "123456789"},
//   "tags": ["a", "b"],
//   ...
// }

// 反序列化
const parsed = superjson.parse(jsonString);
// { createdAt: Date, bigNumber: BigInt, ... }
```

### 5.3.3 项目中的配置

**服务端配置 (init.ts)**

```typescript
const t = initTRPC.create({
  transformer: superjson,
});
```

**客户端配置 (client.tsx)**

```typescript
createTRPCClient<AppRouter>({
  links: [
    httpBatchLink({
      transformer: superjson,
      url: getUrl(),
    }),
  ],
});
```

**Query Client 配置 (query-client.ts)**

```typescript
export function makeQueryClient() {
  return new QueryClient({
    defaultOptions: {
      dehydrate: {
        serializeData: superjson.serialize,
      },
      hydrate: {
        deserializeData: superjson.deserialize,
      },
    },
  });
}
```

### 5.3.4 常见类型转换对照

| JavaScript 类型 | JSON 表现 | Superjson 表现 |
|-----------------|-----------|----------------|
| `Date` | `"2024-01-01T00:00:00.000Z"` | `new Date()` |
| `BigInt` | 不支持 | `BigInt()` |
| `undefined` | 忽略 | `undefined` |
| `NaN` | `null` | `NaN` |
| `Infinity` | `null` | `Infinity` |
| `Set` | `{}` | `['a', 'b']` |
| `Map` | `{}` | `{k1: v1}` |

### 5.3.5 注意事项

**避免在转换后的数据上直接修改：**

```typescript
// ❌ 错误做法
const result = superjson.parse(jsonString);
result.createdAt.setFullYear(2025); // 可能导致问题

// ✅ 正确做法
const result = superjson.parse(jsonString);
const newDate = new Date(result.createdAt);
newDate.setFullYear(2025);
```

## 5.4 错误处理最佳实践

### 5.4.1 TRPCError 标准错误码

```typescript
import { TRPCError } from '@trpc/server';

throw new TRPCError({
  code: "BAD_REQUEST",           // 400 - 请求参数错误
  message: "Invalid input",
});

throw new TRPCError({
  code: "UNAUTHORIZED",          // 401 - 未登录
  message: "Please login first",
});

throw new TRPCError({
  code: "FORBIDDEN",             // 403 - 无权限
  message: "Access denied",
});

throw new TRPCError({
  code: "NOT_FOUND",             // 404 - 资源不存在
  message: "Resource not found",
});

throw new TRPCError({
  code: "TIMEOUT",               // 504 - 请求超时
  message: "Request timeout",
});

throw new TRPCError({
  code: "INTERNAL_SERVER_ERROR", // 500 - 服务器错误
  message: "Something went wrong",
});
```

### 5.4.2 项目中的错误处理示例

**资源不存在**

```typescript
const voice = await prisma.voice.findUnique({ ... });

if (!voice) {
  throw new TRPCError({
    code: "NOT_FOUND",
    message: "Voice not found",
  });
}
```

**权限不足**

```typescript
const hasActiveSubscription = (customerState.activeSubscriptions ?? []).length > 0;

if (!hasActiveSubscription) {
  throw new TRPCError({
    code: "FORBIDDEN",
    message: "SUBSCRIPTION_REQUIRED",
  });
}
```

**前置条件不满足**

```typescript
if (!voice.r2ObjectKey) {
  throw new TRPCError({
    code: "PRECONDITION_FAILED",
    message: "Voice audio not available",
  });
}
```

**外部 API 调用失败**

```typescript
const { data, error } = await chatterbox.POST("/generate", { ... });

if (error || !(data instanceof ArrayBuffer)) {
  throw new TRPCError({
    code: "INTERNAL_SERVER_ERROR",
    message: "Failed to generate audio",
  });
}
```

### 5.4.3 前端错误处理

```typescript
const deleteMutation = useMutation(
  trpc.voices.delete.mutationOptions({
    onSuccess: (data) => {
      // 成功处理
      toast.success('删除成功');
    },
    onError: (error) => {
      // 错误处理
      switch (error.data?.code) {
        case 'NOT_FOUND':
          toast.error('资源不存在');
          break;
        case 'FORBIDDEN':
          toast.error('无权限操作');
          break;
        default:
          toast.error('删除失败，请重试');
      }
    },
  })
);
```

### 5.4.4 全局错误处理

可以在 middleware 中添加全局错误处理：

```typescript
const errorHandlerMiddleware = t.middleware(async ({ next, type, path }) => {
  try {
    const result = await next();
    return result;
  } catch (error) {
    // 统一日志记录
    console.error(`[${type}] ${path}:`, error);
    
    // 重新抛出或转换错误
    if (error instanceof TRPCError) {
      throw error;
    }
    
    throw new TRPCError({
      code: "INTERNAL_SERVER_ERROR",
      message: "Internal server error",
    });
  }
});
```
